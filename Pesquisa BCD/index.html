Pesquisa sobre Sistema de Gerenciamento de Banco de Dados (SGBD)

O que é SGBD

SGBD significa Sistema de Gerenciamento de Banco de Dados. É o software responsável por criar, organizar, controlar e acessar os bancos de dados.

Ele funciona como um “bibliotecário digital”, que organiza, encontra e entrega o dado certo quando o usuário solicita.

O SGBD armazena, organiza, consulta, atualiza, protege e garante a integridade dos dados — controlando o acesso e o desempenho de todo o banco de dados.

Exemplos de uso de SGBD

  1. Bancos e aplicativos financeiros: guardam dados de contas, transações, clientes e histórico de transferências.
    → Toda vez que você consulta o saldo ou faz um PIX, o sistema acessa o banco de dados em um SGBD.
  2. Escolas e instituições de ensino: armazenam dados de alunos, professores, notas e presença.
    → Quando o professor lança suas notas no sistema, ele está salvando essas informações no banco de dados via SGBD.
  3. Lojas, supermercados e e-commerces: mantêm informações sobre produtos, preços, estoque e pedidos.
    → Ao adicionar algo ao carrinho, o site registra isso no banco de dados.

Outros exemplos incluem redes sociais, hospitais, empresas de transporte, prefeituras e órgãos públicos.

Tipos de SGBD

SGBD Relacional (SQL)

Organiza os dados em tabelas (linhas e colunas), parecidas com planilhas. Cada tabela se relaciona com outras por meio de chaves (como o ID de um aluno, por exemplo).

  • Vantagens:
    • Ideal para dados bem estruturados
    • Estrutura organizada e padronizada
    • Alta confiabilidade e segurança
  • Desvantagens:
    • Menos flexível para dados variados
    • Pode ser mais lento com grandes volumes não estruturados

SGBD Não Relacional (NoSQL)

Armazena dados de forma mais flexível, sem tabelas fixas, podendo usar formatos como documentos, chave-valor, grafos ou colunas. É rápido e ideal para grandes volumes de dados variados.

  • Vantagens:
    • Alta flexibilidade e escalabilidade
    • Fácil adaptação a mudanças
  • Desvantagens:
    • Menor consistência dos dados
    • Falta de padronização

Características do SGBD

  • Armazenamento estruturado
  • Linguagem de consulta (SQL ou NoSQL)
  • Segurança e controle de acesso
  • Integridade e consistência
  • Backup e recuperação de dados
  • Gerenciamento de múltiplos usuários

Importância no desenvolvimento de sistemas

  • Organização eficiente de informações
  • Maior segurança e confiabilidade
  • Suporte ao desenvolvimento de aplicações
  • Redução de redundância de dados
  • Base para a tomada de decisões
